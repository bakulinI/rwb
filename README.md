# Сервис "Сейчас ищут"

Это небольшой Go-сервис для виджета "Сейчас ищут".

Он читает поисковые события из NATS, хранит их в памяти за последние 5 минут и быстро отдает топ запросов через HTTP.
Что сделано

- Консьюмер NATS, который подписывается на subject `search.queries`.
- HTTP API:
  - `GET /health` - проверка, что сервис жив.
  - `GET /top?limit=10` - получить топ популярных запросов.
  - `POST /stopwords` - заменить стоп-лист без перезапуска сервиса.
- Окно данных 5 минут.
- Кэшированный топ: на чтение сервис просто отдает уже готовый список, поэтому ручка `/top` работает быстро.
- Unit-тесты основной бизнес-логики.
- Dockerfile и docker-compose для локального запуска вместе с NATS.

Что сделано дополнительно

- Стоп-лист. Слова можно задать через переменную окружения `STOP_WORDS`, например `casino,spam`, или заменить через `POST /stopwords`.
- Простая защита от накрутки. Один источник не может добавить один и тот же запрос в топ больше `MAX_PER_SOURCE_IN_WINDOW` раз за 5 минут. По умолчанию лимит равен 5.
- Нормализация запроса: пробелы по краям убираются, регистр приводится к нижнему.

Контракт события в брокере
Смежный сервис поиска должен отправлять в NATS JSON такого вида:

```json
{
  "query": "iphone 15", - поисковый запрос, обязательное поле.
  "user_id": "user-123", - идентификатор пользователя, если он есть.
  "ip": "127.0.0.1", - IP адрес, если пользователя нельзя определить.
  "timestamp": "2026-05-23T12:00:00Z" - время события. Если поле не передали, сервис возьмет текущее время.
}
```

=
Для защиты от накрутки сервису нужен источник события. Сначала используется `user_id`, если его нет - `ip`, если нет и его - источник считается `unknown`.

Локальный запуск

```bash
docker compose up --build
```

После запуска API будет доступно на `http://localhost:8080`.

Проверить здоровье:

```bash
curl http://localhost:8080/health
```

Отправить тестовое событие в NATS можно так:

```bash
docker compose run --rm nats-box nats pub --server nats://nats:4222 search.queries '{"query":"phone","user_id":"u1"}'
docker compose run --rm nats-box nats pub --server nats://nats:4222 search.queries '{"query":"phone","user_id":"u2"}'
docker compose run --rm nats-box nats pub --server nats://nats:4222 search.queries '{"query":"laptop","user_id":"u3"}'
```

Получить топ:

```bash
curl "http://localhost:8080/top?limit=10"
```

Пример ответа:

```json
[
  {
    "query": "phone",
    "count": 2
  },
  {
    "query": "laptop",
    "count": 1
  }
]
```

Заменить стоп-лист:

```bash
curl -X POST http://localhost:8080/stopwords \
  -H "Content-Type: application/json" \
  -d '["phone", "casino"]'
```

Запуск тестов

```bash
go test ./...
```

Почему без базы данных - по условию нужен топ только за последние 5 минут. Поэтому проще и быстрее держать события в памяти процесса. В реальном продакшене можно добавить Redis.
